## 풀이 : DFS
> 14888번과 매우 유사하다

### 14888과 다른점
```
연산자의 개수가 제한적이지 않다.
-> dfs기반의 풀이를 작성할 때 함수의 return지점이 명확해야한다.
return을 설정하지 않으면 다음 if문으로 넘어가 index error를 유발할 수 있다.
왜 14888보다 난이도가 낮은 s2로 책정된 지 모르겠다.
```

## Code 
```py
N = int(input())
operands = list(map(int, input().split()))
operators = list(map(int, input().split()))

v_max = -10e9
v_min = 10e9

def solve(depth, rax):
    global v_max, v_min
    if depth == N:
        v_max = max(v_max, rax)
        v_min = min(v_min, rax)
        return
    
    if operators[0]:
        operators[0] -= 1
        solve(depth+1, rax+operands[depth])
        operators[0] += 1

    if operators[1]:
        operators[1] -= 1
        solve(depth+1, rax-operands[depth])
        operators[1] += 1

    if operators[2]:
        operators[2] -= 1
        solve(depth+1, rax*operands[depth])
        operators[2] += 1
        
    if operators[3]:
        operators[3] -= 1
        solve(depth+1, 0-(abs(rax)//abs(operands[depth])) if (rax<0 and operands[depth]>0) else rax//operands[depth])
        operators[3] += 1

solve(1,operands[0])
print(v_max)
print(v_min)
```
